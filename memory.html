Memory System

MARK's persistent local memory that remembers information forever with 100% local storage.

What is the Memory System?

MARK's memory system is its standout feature. If you tell MARK something, it remembers it forever (saved locally on your machine). Unlike other AI assistants that forget after each session, MARK maintains a persistent memory of your preferences, facts, and information.

Privacy First

All memories are stored locally on your machine using a SQLite database. No data is sent to external servers. Your personal information remains completely private and secure.

How the Memory System Works

1

Storage

MARK uses a local SQLite database to store your memories. Each memory has a key-value structure with metadata for organization.

2

Categorization

The system intelligently categorizes information based on content. For example, "my name is John" gets categorized under "name".

3

Retrieval

When you ask a question, MARK retrieves relevant memories and includes them in the AI context for personalized responses.

4

Context

Memories are seamlessly integrated into conversations without you having to recall previous information.

Memory Commands

Natural Language Commands

You can use natural language to store memories in multiple languages:

English

remember that my name is John
remember that I prefer Python for scripting
remember that I live in New York

Italian

ricorda che mi chiamo Marco

ricorda che amo il caffè

Management Commands

Command Description
/memory list View all saved memories
/memory search <term> Semantic search in memory
/memory delete <id> Delete a specific memory by ID
/memory clear Delete all memories

Practical Examples

Personal Preferences

After telling MARK your preferences once, it remembers them for all future interactions:

# You say once:
> remember that I prefer Python for scripting

# Later, when you ask:
> write a script to sort a list

# MARK responds with Python code automatically because it remembers your preference!

Project Information

Store details about ongoing projects to maintain context:

# At the start of a project:
> remember that I'm working on a web scraping project in Python
> remember that the project is called "DataHarvester"
> remember that the target website is https://example.com

# Later, when you ask for help:
> how do I handle rate limiting in my project?

# MARK knows you're referring to the "DataHarvester" project!

Personal Facts

Maintain personal information for personalized assistance:

# Store personal information:
> remember that my name is Sarah Johnson
> remember that I live in Seattle, WA
> remember that I work as a data scientist
> remember that I have a dog named Max

# When asking for location-based information:
> what's the weather forecast?

# MARK can provide Seattle-specific information!

Security & Privacy

Local Storage

All memories are stored locally on your machine using SQLite. No data leaves your computer.

Encrypted Keys

API keys are stored securely using your OS's native keychain (Windows Credential Manager, macOS Keychain, etc.).

No Data Sharing

MARK never shares your personal information with third parties or external services.
